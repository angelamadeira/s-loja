Design System · Studio Suzu

Companion do Brandbook Studio Suzu para o produto digital: mesma voz, mesma tipografia, mesma cor — aqui aplicadas a tokens e componentes de código. Fonte única da verdade: esta página não copia valores, ela lê css/tokens.css e css/components.css ao vivo. Os componentes abaixo são os mesmos elementos da loja, renderizados com as classes reais.

Regra do DS: cor só existe como hex dentro de css/tokens.css. Em qualquer outro lugar — incluindo esta própria página — só var(--token). Os valores hex mostrados nos swatches abaixo são lidos do navegador em tempo real via getComputedStyle, nunca escritos aqui.

Fundamentos · 01

Cores

Tokens definidos uma única vez em :root, em duas camadas — paleta (bruta, nunca troca de tema) e semântico (papéis, ganha override no tema escuro da 0.5b). Agrupados por função — não por matiz — porque é assim que são usados no CSS.

Tamanhos (PDP) — .szgrid

Grade de tamanho da PDP: até três colunas fixas (Pequeno/Médio/Grande), cada .sz já mostra nome, medida e preço daquele tamanho — sem precisar escolher pra ver o valor. Tamanho esgotado é .sz.out: opacidade reduzida e o preço vira "esgotado", a mesma lógica de .fcsz.out (Chips de tamanho), só que no card inteiro do tamanho.

Sistema · 05

Espaçamento

Escala canônica base-4 (--space-1 a --space-8, de 4 a 64px). É uma escala nova em tokens.css — a adoção é incremental: o CSS legado da loja continua com valores ad-hoc (margens/paddings soltos), e migra componente por componente conforme cada tela é revisada, nunca de uma vez só.

Sistema · 06

Raio

Escala canônica de border-radius (--r-sm a --r-2xl, mais --r-pill para pílula e --r-round para círculo). Mesma regra de adoção incremental do espaçamento: componentes existentes com raio "solto" migram aos poucos, sem trocar pixel sem revisão.

Sistema · 07

Movimento

Três curvas (--ease, --ease-out, --ease-pop) e três durações (--dur-fast, --dur, --dur-slow) — canônicas, adoção incremental sobre as transições ad-hoc já existentes no CSS legado. Princípio: micro-interações usam curva + duração rápidas, entradas de tela usam durações mais longas, e --ease-pop é reservado só para elementos que "aparecem" (leve overshoot) — nunca para deslocamento comum.

Curvas × duração

O ponto que desliza usa animation-timing-function e animation-duration lidos ao vivo de cada token — nunca uma curva reescrita aqui.

--ease-out + --dur-fast
micro-interações — hover, toggle, chip
--ease + --dur
padrão — a maioria das transições
--ease-pop + --dur-slow
só para "aparecer" — overshoot leve

prefers-reduced-motion: reduce está ativo neste navegador — as animações acima estão desligadas.

Marca · 08

Logo / Selo

O selo é a marca gráfica do Studio Suzu — mesmos <path> #p-su/#p-zu usados em index.html, sem duplicar arte fora dos <defs> compartilhados no topo desta página.

Sempre (regras do Brandbook)

  1. O selo com contorno (outline) é o logo oficial — tinta (--ink) no tema claro, branco (--white) no tema escuro.
  2. Espaço livre mínimo ao redor: ¼ do diâmetro do selo, nos 4 lados.
  3. Tamanhos mínimos: 32px outline · 24px preenchido · 80px em uso inline (ao lado do texto).
  4. Abaixo de 32px, sempre trocar para o selo preenchido — o traço fino do outline deixa de ler.
  5. Sobre padrão (seigaiha/shippō), o selo sempre vive dentro de uma clareira sólida — nunca direto sobre a estampa.
  6. Traço do anel: proporção fixa de 52/1600 (mesma relação usada no viewBox e stroke-width do .seal abaixo).
  7. Letras só em --ink ou --white — nunca outra cor.
  8. Preenchimento sempre chapado — sem gradiente, sem sombra.
  9. Cor de sabor da paleta preenchendo o selo só existe no flavor system — nunca em uso institucional.
  10. Wordmark exatamente como fornecido — nunca esticar, inclinar ou distorcer.

Marca · 09

Ícones

Família Feather: viewBox 0 0 24 24, fill:none, stroke:currentColor, stroke-width:2, stroke-linecap/stroke-linejoin:round, tamanho ~1.2em. Marca · 10

Padrões

Duas estampas autorais — seigaiha (ondas, leques de arcos concêntricos) e shippō (círculos entrelaçados) — mesmos <pattern> de index.html, reaproveitados aqui via os <defs> do topo da página.
